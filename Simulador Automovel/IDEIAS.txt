Um rato de bola (antigo, nao laser) costuma ter 2 Optical Rotary Encoders. 
(o Scroll button é um rotary encoder mas quase sempre mecanico)
Para ligar ao Arduino o componente era suposto ter um LM393 a amplificar e quadrificar, mas este simples esquema e script mostram que é possivel mesmo sem isso (e nos comments há um tipo de dá montes de boas ideias para usar rotary encoders magneticos):
http://www.electroschematics.com/10494/arduino-optical-position-rotary-encoder/
(mas resta o problema desses sensores só terem 3 pinos no led e 3 pinos no sensor... como sei a direção?)
Outra forma é ligar o rato ao Arduino via PS2 e usar este script (espero que não seja muito pesado para o Arduino):
https://playground.arduino.cc/ComponentLib/Ps2mouse
https://playground.arduino.cc/uploads/ComponentLib/mouse.txt
Ourta forma é ligar o rato ao PC, mas o windows nao o pode usar, e nós temos de usar WM_INPUT, só encontro codigo em C++:

...mas converte-se bem para vb.net, com <DllImport fica limpinho e muito poderoso:
